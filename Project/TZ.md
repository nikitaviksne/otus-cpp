# Цель проекта

Разработать легковесную, потокобезопасную библиотеку для реализации паттерна «Наблюдатель» (Observer) с использованием современных стандартов C++ (17/20). Библиотека должна позволять объектам взаимодействовать друг с другом без прямой связности.

# Функциональные требования

1. Типобезопасность: Сигнал должен жестко задавать типы передаваемых аргументов через шаблоны.
2. Поддержка различных типов слотов:
   1. Свободные функции.
   2. Лямбда-выражения (в том числе с захватом контекста).
   3. Методы классов (указатель на экземпляр + указатель на метод).
3. Управление жизненным циклом (Auto-disconnect):
   1. При уничтожении объекта-слушателя все связанные с ним слоты должны автоматически отключаться.
   2. Реализовать базовый класс (например, Trackable) или механизм на основе std::weak_ptr.
 4. Многопоточность:
   1. Безопасная подписка и отписка из разных потоков.
   2. Возможность выбора режима вызова (синхронный в потоке отправителя или через очередь событий — опционально).
# Технические требования (Stack)
- Стандарт: C++17 или C++20.
- Система сборки: CMake.
-  Зависимости: STL, GTest (для тестов), Google Benchmark (для оценки производительности).
- Метапрограммирование: Использование Variadic Templates, Perfect Forwarding, SFINAE или Concepts.
# Критерии успеха (Definition of Done)
   Проект считается успешно выполненным, если он соответствует следующим критериям:
1. Техническая реализация (Code Quality)
- Zero Memory Leaks: Отсутствие утечек памяти при массовой подписке/отписке (проверяется Valgrind/ASan).
- Type Safety: Попытка соединить сигнал со слотом неверного типа приводит к ошибке компиляции, а не к падению в рантайме.
- Header-only (желательно): Библиотека удобна в подключении (только заголовочные файлы).
2. Тестирование
-  Code Coverage: Покрытие unit-тестами не менее 80-90% логики.
-  Тестовые сценарии:
   1. Вызов сигнала с 0, 1, 5 аргументами разных типов.
   2. Автоматическая отписка при выходе объекта-слушателя из области видимости.
   3. Поведение при рекурсивном вызове (сигнал вызывает слот, который вызывает тот же сигнал).
3. Архитектурная чистота
-  Отсутствие дублирования кода за счет использования шаблонов.
-  Минимальный оверхед (сравнение скорости вызова со стандартным std::function).
4. Документация
-  Наличие README.md с примерами использования (Quick Start).
-  Краткое описание архитектуры: как реализовано стирание типов (Type Erasure) и как работает механизм отслеживания времени жизни.
-  Видео пример работы приложения.
