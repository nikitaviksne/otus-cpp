# Основные требования
1. Типобезопасность Сигнал будет жестко задавать типы передаваемых аргументов через шаблон.
2. Поддержка различных типов слотов. Обычные функции, ламбда-функции, методы классов.
3. Если уничтожается объект-слушатель, все связанные с ним слоты автоматически будут отвязываться.
4. Потокобезопасность. Спроектировать библиотеку потокобезопасной. Реализовать возможность подписки и отписки из разных потоков.

# Архитектурные особенности

Использован интрузивный список и SBO (Small Object Optimization). 

Пользователю следует избегать циклических зависимостей между сигналами во избежание deadlock

# Запуск сборки и тестов
## С использованием docker
```
docker buildx build -t cpp-dev --progress=plain --load .
```

## Без Docker
```
cmake -B build -S . && cmake --build build && cmake --build build --target test && ./build/run_benchmarks
```
